# Claude Code Workflow Rules for PhonoMorph

## Auto-Approve These Commands (No Confirmation Needed)

### Index & Build Operations
- `npm run rebuild-index` — Deterministic rebuild of public/data/index.json and shards
- `/tmp/add-transformation.sh <shift_id> "<message>"` — Complete rebuild+commit pipeline

### Git Operations (Transformation Files Only)
- `git add public/data/transformations/*.json` — Adding new transformation files
- `git add public/data/symbols/*.json` — Adding new symbol files
- `git add public/data/index.json` — Adding rebuilt index
- `git commit -m "feat: add ..."` — Commits for new transformations
- `git commit -m "mark: add ... to unattested"` — Marking unattested shifts
- `git log` / `git status` / `git diff` — Read-only operations

### File Operations
- Creating files in `/tmp/` — Temporary scripts and scratch work
- Writing transformation JSON files to `public/data/transformations/`
- Writing symbol files to `public/data/symbols/`

---

## Manual Approval Required

- `git push` — Always ask before pushing to remote
- `git reset` / `git rebase` / `git checkout --` — Destructive operations
- Modifying existing transformation files (use `/improve` skill instead)
- Creating/deleting branches
- Any operation affecting `src/` (frontend code)

---

## Typical Session Pattern (Single Approval)

When asked to fill multiple transformations:
> "Create transforms [list], rebuild, and commit — one approval for the whole chain"

Example batching:
```
Create y_to_ash → /tmp/add-transformation.sh y_to_ash "feat: add y_to_ash"
Create eps_to_caret → /tmp/add-transformation.sh eps_to_caret "feat: add eps_to_caret"
Create ash_to_y → /tmp/add-transformation.sh ash_to_y "feat: add ash_to_y"
```

All three rebuild+commit cycles auto-approved as a group.

---

## Notes

- Transformation file creation (Write tool) is always safe — no approval needed
- Index rebuilds are idempotent and safe — no approval needed
- The `/tmp/add-transformation.sh` script prevents repetitive approval chains
- For research/fill batches, group similar shifts and process them together
